# Vlog / 视频素材整理通用方法

## 目标

在正式粗剪前，把大量原始素材整理成一张可检索、可筛选、可进入剪映 / Premiere / Final Cut 的素材表。

核心不是逐条“描述视频”，而是帮助剪辑时快速回答三个问题：

1. 这个素材是什么？
2. 它属于哪个场景组，能和哪些素材互相替换？
3. 它有没有可用口播、关键信息或情绪点？

## 输入信息

每个项目开始前，先准备这些信息：

- 原始素材文件夹
- 项目主题，例如“韩国邮轮 Vlog”“东京三日游”“周末探店”
- 大致行程 / 时间线
- 重要地点、人名、品牌、专有词
- 最终剪辑平台，例如剪映、Premiere、Final Cut
- 输出位置，例如本地 HTML 预览、飞书文档、Notion、表格文件

## 输出表字段

| 字段 | 用途 |
|---|---|
| 候选素材名称 | 原始文件名，作为素材 key |
| 拍摄时间 | 用于按真实时间线排序 |
| 视频时长 | 判断素材使用成本和节奏 |
| 大场景 / Set | 标记同一场景、同类镜头、可替换素材 |
| 口播内容 | 清晰口播保留原话，不清晰时写摘要，低可信留空 |
| 代表截图 | 方便快速扫图选素材 |

## Set 分组规则

Set 是这套方法的核心。

同一个 Set 里的素材表示：

- 同一地点
- 同一动作
- 同一话题
- 同一段口播的多次尝试
- 同一画面的不同角度 / 景别 / 重拍

剪辑时通常只需要从同一 Set 中选 1 条或少量几条，不要全部用。

示例：

| Set | 含义 | 剪辑处理 |
|---|---|---|
| Set 001：抵达码头 | 单独建立镜头 | 通常必选 |
| Set 002：登船前自拍视频 | 多条重复口播 | 选最自然的一条 |
| Set 003：行李托运 | 同一流程多角度 | 选 1-2 条组合 |
| Set 004：海边日落 | 氛围镜头 | 可按画面美感择优 |

## 口播处理原则

口播不做“盲目逐字稿”，而做“剪辑可用文本”。

优先级：

1. 清晰近距离口播：尽量保留原话，修正标点和明显错词。
2. 多人对话 / 环境嘈杂：整理成摘要式口播。
3. 远处人声 / 背景广播 / 音乐：除非对剧情重要，否则留空。
4. AI 明显幻觉：直接删除，不要为了填表强行保留。

校正口播时要结合 context：

- 项目背景
- 当前 Set 场景
- 画面内容
- 已知地点 / 人名 / 品牌 / 行程词

## 截图规则

截图只服务于快速判断素材，不追求每条都多图。

推荐规则：

- 短视频、画面变化小：1 张代表截图。
- 长运镜、前后画面差异大：2-3 张截图。
- 同一人物同一角度重复口播：1 张即可。
- 场景切换明显：多放截图，避免误判内容。

在线文档中，截图应尽量：

- 直接放进表格，而不是只写图片文件名。
- 保持原始比例，不强行裁切为横图。
- 如果表格图片容器出现异常空白，优先尝试写入图片宽高；仍异常时，考虑把截图从表格单元格移到素材条目下方。

## 推荐流程

1. 扫描素材
   - 读取文件名、拍摄时间、视频时长。
   - 按拍摄时间排序。

2. 抽代表帧
   - 每条素材至少抽 1 张代表图。
   - 对长视频或画面变化大的素材抽多张。

3. 初步识别场景
   - 根据画面、时间、文件相邻关系，生成初始 Set。
   - 同一场景重复拍摄的素材归入同一 Set。

4. 识别口播
   - 先做自动识别。
   - 再用项目背景和画面内容校正。
   - 不可信内容留空。

5. 生成本地预览
   - 先用 HTML 或表格看整体效果。
   - 检查字段、截图大小、Set 是否合理。

6. 写入在线文档
   - 确认本地预览没问题后，再更新在线文档。
   - 大表建议分段写入，避免接口超时。

7. 粗剪衔接
   - 按 Set 作为候选素材组。
   - 每个 Set 选 A/B/C 候选。
   - 口播列用于决定是否保留原声、做字幕或作为旁白信息。

## 剪辑使用方式

粗剪时可以按这个逻辑处理：

1. 先按时间顺序浏览 Set。
2. 每个 Set 只选最合适的候选素材。
3. 对有口播的素材，优先判断：
   - 这句话是否推动信息？
   - 是否有情绪价值？
   - 画面是否也足够好？
4. 对无口播但画面好的素材，用作转场、氛围或 B-roll。
5. 同一个 Set 内避免重复使用多个相似镜头，除非需要节奏蒙太奇。

## 质量标准

一张合格的素材整理表应该做到：

- 按时间线能看懂旅程发生顺序。
- 看 Set 就知道哪些素材可以互相替换。
- 看截图就能快速回忆画面。
- 看口播就能判断是否值得保留原声。
- 低可信文本不会污染剪辑判断。

## 项目启动模板

```text
项目名称：
素材文件夹：
视频主题：
大致行程：
重要地点 / 人名 / 专有词：
最终剪辑平台：
输出形式：

整理要求：
1. 按拍摄时间排序。
2. 按 Set 标记同一场景和可替换素材。
3. 每条素材保留拍摄时间、视频时长、口播内容、代表截图。
4. 口播采用校正版 / 摘要式，不可信内容留空。
5. 先生成本地预览，确认后再更新在线文档。
```
